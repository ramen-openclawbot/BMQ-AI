# BMQ-AI — Claude Instructions

## Quy trình làm việc

### 1. Luôn đặt câu hỏi làm rõ trước khi làm
Trước khi bắt đầu bất kỳ task nào, phải đặt câu hỏi để hiểu đúng yêu cầu. Không tự suy đoán nếu còn điểm chưa rõ.

### 2. Trình bày plan trước — chờ approve
Sau khi hiểu yêu cầu, trình bày **plan chi tiết** (các bước sẽ làm, file sẽ thay đổi, logic chính) rồi hỏi:
> "Anh có approve plan này không?"

Không được bắt đầu code khi chưa có approve.

### 3. Code và commit chỉ khi được approve
- **Approve** → tiến hành code, sau đó commit với message rõ ràng.
- **Không approve** → hủy, hỏi lại để điều chỉnh plan.
